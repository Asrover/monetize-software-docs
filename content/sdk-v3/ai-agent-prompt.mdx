# AI agent prompt ‚Äî hand the integration to an agent

Integrating SDK 3.0 into someone else's project and want to delegate the chores to a coding agent (Cursor, Claude Code, Copilot, Windsurf, Cline)? This page gives you a **ready-made prompt + a single link** the agent needs for a full-stack integration.

<Callout type="info">
  **TL;DR.** Copy the prompt below, fill in your `paywallId` / `apiOrigin` / channel and paste it to the agent. It will fetch [/llms-sdk-v3.txt](https://monetize.software/llms-sdk-v3.txt), read the focused pack (~190 KB) and propose a plan + code.
</Callout>

## What is `llms-sdk-v3.txt`

It's a **focused dump** of SDK 3.0 docs in one `.txt` file following the [llmstxt.org](https://llmstxt.org) standard. The agent pack contains **only** SDK 3.0‚Äìrelevant material:

- Full SDK 3.0 reference: `AuthClient`, `BillingClient`, `PaywallUI`, `ApiGatewayClient`, bootstrap, events, errors, storage adapters, localization, security model.
- Three end-to-end integration guides: **Web (SPA)**, **Chrome Extension (MV3)**, **Headless (server / custom UI)**.
- Paywall features: anonymous sign-in, trial, tokenization, translations, local pricing.
- Setup overviews: payment processors, custom domains, webhooks.

The file is regenerated on every doc deploy and is always current. The size is calibrated to fit in any modern model's context (Claude / GPT-4o / Gemini) with headroom for the project code.

## Step 1. Gather parameters

Before launching the agent, have these ready:

| Parameter      | Where to get it                                                  | Example                          |
| -------------- | ---------------------------------------------------------------- | ------------------------------- |
| `paywallId`    | Dashboard ‚Üí **Paywalls** ‚Üí open the relevant one, ID is in the URL | `3`                             |
| `apiOrigin`    | Your paywall's [custom domain](/docs-v2/custom-domains)          | `https://billing.theirapp.com`  |
| Channel        | `web` / `extension` / `headless`                                 | `web`                           |
| `providerId`   | (optional) Dashboard ‚Üí **API Providers**, for the AI proxy       | `8b3f...-uuid`                  |
| Auth needs     | Login required? If yes ‚Äî which providers (Google, Apple)?        | `email + Google OAuth`          |

<Callout type="warning">
  **`apiOrigin` is mandatory** at SDK init and has no fallback. If you haven't connected a custom domain yet ‚Äî do that first, [instructions](/docs-v2/custom-domains/create-domain). Until then the integration won't work.
</Callout>

## What NOT to expect from the agent

- **Creating a paywall in the dashboard.** Manual step ‚Äî the agent doesn't have access to your monetize.software account.
- **Configuring a payment processor.** Manual too: Stripe keys, Paddle vendor ‚Äî added via the dashboard UI.
- **Connecting a custom domain.** Done once on your side ([instructions](/docs-v2/custom-domains/create-domain)). Once DNS is live, the agent gets a ready `apiOrigin`.
- **Real payment testing.** Use [test IDs and test mode](/docs-v2/payment-processor/overview) ‚Äî the agent can suggest which cards, but verification is on you.

## If the integration doesn't work

Give the agent this follow-up:

```text copy
The integration builds but <describe the issue: "paywall.open() shows nothing" /
"401 on bootstrap" / "Stripe checkout returns 400">.

Re-check against https://monetize.software/llms-sdk-v3.txt:
1. The "Errors" and "Security" sections ‚Äî common apiOrigin / CORS issues.
2. The "Events" section ‚Äî could an event be firing before we subscribed?
3. Does the `paywallId` match what's in the dashboard?

If the problem is in my config (not the code) ‚Äî tell me explicitly what I should check
in the dashboard / payment processor.
```
