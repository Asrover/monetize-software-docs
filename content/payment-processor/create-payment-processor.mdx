# Create Payment Processor

Step-by-step guide to creating and configuring a payment processor for handling payments in your application.

<Callout>
  **What is a payment processor?** It's an intermediary between your application
  and payment systems (Stripe/Paddle) that processes transactions and manages
  subscriptions. [Read mode](/docs-v2/payment-processor/overview)
</Callout>

<Steps>

### Step 1: Creating a New Payment Processor

Navigate to the "Payment Processors" section and click the "New Payment Processor" button.

<div style={{ marginTop: "2rem", marginBottom: "2rem" }}>
  ![Create payment processor](/docs/pp-create.png)
</div>

### Step 2: Choose Payment System

Select one of the supported payment systems:

<Tabs items={["Stripe", "Paddle", "Freemius"]}>
  <Tabs.Tab>
    **Stripe** Popular payment system with wide support for cards and payment
    methods. **Advantages:** - Support for most bank cards - Global coverage -
    Simple integration - Detailed analytics **Best for:** - International
    projects - SaaS applications - Web services
  </Tabs.Tab>
  <Tabs.Tab>
    **Paddle** Payment system specializing in digital products and
    subscriptions. **Advantages:** - Built-in tax handling - Support for
    multiple currencies - Specialization in digital goods - Global tax
    compliance **Best for:** - Digital products - Software - Online courses
  </Tabs.Tab>
  <Tabs.Tab>
    **Freemius** Merchant of Record for software businesses with built-in
    licensing, taxes and affiliate management. **Advantages:** - Merchant of
    Record (handles VAT/sales tax) - Built-in license management - Hosted
    checkout with global payment methods - Affiliate platform out of the box
    **Best for:** - WordPress plugins/themes - SaaS products - Desktop apps
  </Tabs.Tab>
</Tabs>

    **One acquirer ↔ one paywall.** Each Freemius acquirer is allowed to be connected to a single paywall at a time (a database trigger enforces this). The acquirer references one Freemius plan, and the prices on the paywall (monthly / annual / lifetime) are routed to the matching billing cycle of that plan automatically. Switching the acquirer to a different paywall later is safe — historical subscriptions keep flowing into their original paywall via the webhook handler.

    **Plans and pricing:** Plan/pricing creation is gated behind Freemius developer-scope canonical signing (not exposed via Bearer Token). Create plans in the Freemius dashboard manually and configure their monthly / annual / lifetime prices to match the prices you set on your paywall.

    **Webhook setup:** Freemius does not expose webhook listener management in the public API either. After creating the payment processor, set the webhook up manually in **Dashboard → your product → Webhooks → Listeners → Add Webhook** with the URL shown in the form and subscribe to: `subscription.created`, `subscription.cancelled`, `subscription.renewal.failed`, `subscription.renewal.failed.last`, `install.trial.started`, `license.expired`, `payment.created`, `payment.refund`.

### Step 5: Save

After filling in all fields:

1. **Key validation**: The system will automatically verify the correctness of API keys
2. **Webhook creation**: A webhook for event processing will be created automatically
3. **Activation**: The processor will be activated and ready to use

</Steps>

## Troubleshooting

<Accordion>
  <Accordion.Item title="Stripe key validation error">
    Check: - Correctness of copied keys - Key compliance with selected mode
    (test/live) - Access permissions in Stripe account
  </Accordion.Item>

  <Accordion.Item title="Paddle key validation error">
    Make sure that: - API Key is created with "Never Expires" and full
    permissions - Vendor ID contains only numbers - You're using the correct
    environment (sandbox/production)
  </Accordion.Item>

  <Accordion.Item title="Freemius key validation error">
    Make sure that: - Product ID is numeric and matches the product in your
    Freemius dashboard - API Token is generated under Settings → API Token for
    the same product - Public/Secret keys correspond to the selected mode
    (Sandbox keys for test, Production keys for live) - The API Token has not
    been revoked
  </Accordion.Item>
</Accordion>
