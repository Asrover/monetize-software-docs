# Quickstart

Start monetizing your application in 10 minutes. Choose the right SDK and follow step-by-step instructions for quick integration.

## Choose the Right SDK

The platform provides four integration approaches to suit your needs:

| Mode                                                  | Advantages                                                                                                                                                | Perfect for                                                                                            | Notes                                                                              |
| ----------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------ | ---------------------------------------------------------------------------------- |
| **SDK v3** — Recommended for new projects             | Bundled npm package; No iframe (Shadow DOM); Headless mode for server-side; Chrome Extension MV3 support out of the box; Modular (`/core`, `/ui`, `/full`) | Modern web apps, SPAs, Chrome / Edge MV3 extensions, Telegram Mini Apps, server-side billing            | Requires a bundler (npm pkg, no CDN)                                               |
| **SDK v2** — Client-Side legacy                       | Single `<script>` tag, no build step; iframe modal; works on plain HTML landings                                                                          | Landing pages without a build step, legacy projects already on v2                                       | Will not pass Chrome Web Store review for new extensions                            |
| **Server-Side SDK** — For Custom Solutions            | Full control over user experience; Custom interface design; Complex business logic; Integration with existing systems                                     | Custom platforms; Complex workflows; CRM/ERP integration; Mobile applications                          | Requires more work: you need to build your own interfaces and authentication logic |
| **Hybrid Mode** — Best of Both Worlds                 | Client SDK's ready UI components; Server-side business logic control                                                                                      | Custom backend integration; Teams wanting UI flexibility with backend control                          | Requires more work: you need to build authentication logic                         |

## SDK v3 (recommended)

Bundled npm package, Shadow DOM modal, headless mode for backends. Pick the channel that fits — full step-by-step guides per channel are in the [SDK v3 Quickstart](/docs-v2/sdk-v3/quickstart).

<Steps>

### Step 1: Create a paywall

In the dashboard → **Paywalls** → **New Paywall**, choose **Client Mode**. Configure plans and payment processor. Note the numeric `paywallId` from the URL.

### Step 2: Install the SDK

```bash
pnpm add @monetize.software/sdk
```

### Step 3: Open the paywall (web / SPA)

```ts
import { PaywallUI } from '@monetize.software/sdk/ui';

const paywall = new PaywallUI({
  paywallId: '3',
  apiOrigin: 'https://YOUR_DOMAIN',
  auth: true // built-in email / OTP / OAuth login
});

document.getElementById('upgrade')!
  .addEventListener('click', () => paywall.open());

paywall.on('purchase_completed', () => unlockPremium());
```
